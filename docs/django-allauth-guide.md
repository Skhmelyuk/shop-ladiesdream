# Інструкція з використання django-allauth в проекті ShopDJ

Бібліотека `django-allauth` використовується в цьому проекті для забезпечення повного циклу управління обліковими записами: реєстрація, аутентифікація, відновлення пароля та верифікація електронної пошти.

## 1. Основні налаштування (settings.py)

Всі ключові параметри `django-allauth` знаходяться у файлі `shop/settings.py`. Ось найважливіші з них, які визначають логіку роботи:

```python
# Дозволяє логінитись як за допомогою email, так і за допомогою username
ACCOUNT_LOGIN_METHODS = {'email', 'username'}

# Обов'язкове підтвердження email перед тим, як користувач зможе увійти
ACCOUNT_EMAIL_VERIFICATION = 'mandatory'

# Поля, які обов'язкові при реєстрації
ACCOUNT_SIGNUP_FIELDS = ['email*', 'username*', 'password1*', 'password2*']

# Використання кастомної форми для реєстрації з додатковими або зміненими полями
ACCOUNT_SIGNUP_FORM_CLASS = 'accounts.forms.CustomSignupForm'

# Куди перенаправляти користувача після успішного входу/виходу або реєстрації
LOGIN_REDIRECT_URL = '/'
LOGOUT_REDIRECT_URL = '/'
ACCOUNT_SIGNUP_REDIRECT_URL = '/'

# Дозволяє виходити з акаунту (Logout) звичайним GET-запитом без проміжної сторінки підтвердження
ACCOUNT_LOGOUT_ON_GET = True
```

## 2. Маршрути (URLs)

У файлі `shop/urls.py` підключені стандартні маршрути `allauth`:

```python
path('accounts/', include('allauth.urls')),
```

Це автоматично створює низку корисних ендпоінтів. Основні з них:
- `/accounts/login/` — Сторінка входу.
- `/accounts/signup/` — Сторінка реєстрації.
- `/accounts/logout/` — Вихід з системи.
- `/accounts/password/reset/` — Відновлення пароля.
- `/accounts/confirm-email/` — Сторінки підтвердження email.

## 3. Підтвердження Email (Email Verification)

Оскільки `ACCOUNT_EMAIL_VERIFICATION = 'mandatory'`, користувачі **не зможуть увійти** у свій акаунт, поки не підтвердять свою електронну пошту. 

Зараз у режимі розробки (`DEBUG = True`) налаштовано виведення всіх листів прямо в консоль терміналу, а не на реальну пошту:
```python
EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend' 
```

**Як тестувати реєстрацію:**
1. Перейдіть на сторінку реєстрації (`/accounts/signup/`) та заповніть форму.
2. Після успішної реєстрації вас перенаправить на сторінку, яка повідомляє про необхідність підтвердити email.
3. Відкрийте термінал, де запущено сервер (`make run`).
4. Знайдіть у логах терміналу повідомлення з посиланням для підтвердження (воно виглядає як `http://127.0.0.1:8000/accounts/confirm-email/унікальний_ключ/`).
5. Скопіюйте це посилання, вставте в браузер і перейдіть за ним.
6. Натисніть кнопку "Підтвердити" на відкритій сторінці. Після цього ви зможете залогінитись.

## 4. Кастомна форма реєстрації

У проекті використовується `CustomSignupForm` (знаходиться у `accounts/forms.py`). 
Ця форма дозволяє додавати власні перевірки (валідацію) паролів або збирати додаткові дані від користувача (наприклад, ім'я, прізвище чи телефон) під час реєстрації, розширюючи стандартну форму `allauth`.

## 5. Шаблони (Templates)

Шаблони для `django-allauth` знаходяться у директоріях `templates/account/` (перевизначення стандартних шаблонів allauth) та частково використовують ваші кастомні сторінки у `accounts/templates/accounts/`. 
Якщо ви хочете змінити зовнішній вигляд сторінок логіну або реєстрації, редагуйте відповідні HTML-файли в цих директоріях.

## 6. Як підключити реальну відправку email на продакшені?

Коли ви будете розгортати проект на реальному сервері, вам потрібно буде змінити `EMAIL_BACKEND` на SMTP у файлі `shop/settings.py` (закоментований блок коду там вже є):

```python
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.gmail.com'
EMAIL_PORT = 587
EMAIL_USE_TLS = True
EMAIL_HOST_USER = 'your_email@gmail.com' 
EMAIL_HOST_PASSWORD = 'your_app_password' # Пароль додатку з налаштувань Google
```
